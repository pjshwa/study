Programs as "Functions"

  • Ignore the details of the computation, rather focus on the result being computed
  • A program is equivalent to a black box which transforms input to output

 

Mathematical Function

  • Definition in mathematical terminology

    • Mapping of elements of some set to some value from a set of values.

 

Function Definition vs. Application

  • Definition

    • Description of how a value is to be computed using formal parameters.
    • square(x) = x * x

  • Application

    • Call to a defined function using actual parameters.
    • y = square(4)

  • Q. Is Declarative Programming paradigm equal to Functional Programming?

    • A. Functional Programming is a representative example of Declarative Programming paradigm.

 

Consequences of Using Functions

  • There is no notion of assignment.

    • x = x + 1 makes no sense!

      • You cannot change the input value in the middle of execution of a function
      • x1 = x + 1 something like this makes sense though.

  • The lack of assignment also

    • Makes loops impossible.
    • Makes functions have no internal states
    • Makes the value of any function cannot depend on the order of evaluation of its arguments
    • Allows us to manipulate functions in arbitrary ways

 

Functional PL does not have loops

  • It uses recursion to perform repetitive tasks

    • No control variable is allowed :(

  • You should pass the internal state itself to the function's input.

 

Referential Transparency

  • The property whereby a function's value depends only on the values of its arguments

  • Highly useful for concurrent programming!

  • Also referred to as value semantics

    • Only thing that affects the computation are values

 

Functions: First-Class Data Values

  • Functions must be viewed as values themselves. They must be general language objects.

  • e.g., composition

    • A function which takes two functions as parameters, and produces another function as its returned value.
    • where
    • Such functions are called higher-order functions
