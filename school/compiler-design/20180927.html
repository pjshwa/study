Regular Expressions (Inductive Definition)

Regular expressions (REs) can be used to describe patterns.

  • A regular expression r is a pattern that describes a set of lexemes
  • Lexemes are made of characters of a finite set called alphabet.

 

Writing Regular Expressions

Write a regular expression for each of the following sets of tokens:

  1. Ruby binary literals consisting of 0b followed by the binary number.

    • Examples: 0b001011

  2. Ruby binary literals, with an optional underscore ("_") between a pair of binary digits.

    • Examples: 0b0_101 but not 0b_1, 0b1

  3. Floating point literals

  4. Floating point literal with scientific notation

    • Examples: 1.2e-2, 2.3E+34, 2.3E34.

Relationship between NFAs and DFAs

  • DFA is a special case of an NFA

    • DFA has no transitions

    • DFA's transition function is single-valued

      • not possible to have 2 transitions from one state on a same symbol

  • NFA can be simulated with a DFA (less obvious?)

 

Summary: NFAs and DFAs

A FA is a DFA if

  • It has no transitions and the transition function is single-valued

 

Automating Scanner Construction

To convert a spec to code:

  1. Write down the RE for the input language
  2. Build a big NFA
  3. Build the DFA that simulates the NFA (the interesting step)
  4. Minimize the number of states in the DFA
  5. Generate the scanner code

 

Thompson's Construction (RE → NFA)

Key idea:

  • construct NFA for each symbol & operator of RE
  • Join them with moves in precedence order of RE operators
